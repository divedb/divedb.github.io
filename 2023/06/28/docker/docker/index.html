    退出容器

    • exit等同于Ctrl+D

    • Ctrl+p+q

    在 Docker 中，exit 和 Ctrl + P + Q 是两种不同的方式来退出容器的交互模式，它们有一些区别：

    1. exit 命令：

      • 当你在容器内输入 exit 命令时，它会终止当前 shell 进程，导致容器停止并退出。
      • 这个操作将结束容器的主进程，因此整个容器会停止运行。

    2. Ctrl + P + Q 组合键：

      • 使用 Ctrl + P + Q 组合键，你将从容器的交互模式中脱离出来，但容器本身并不会停止。
      • 这种方式实际上是将当前终端的输入焦点从容器移开，使你回到主机的 shell。容器将继续在后台运行。

      或者在启动容器时使用 -d 选项：

      这种方式适用于在容器中运行一个服务，同时仍然需要访问主机的终端。

    总的来说，如果你希望停止容器并退出，可以使用 exit 命令。如果你只是想退出容器的交互模式而不停止容器，可以使用 Ctrl + P + Q 组合键。

    1. 什么是 Manifest？

    在 Docker 中，manifest 是一个描述镜像的元数据文件，它包含了镜像的架构（如 amd64、arm64）、操作系统（如 linux、windows）以及镜像的层信息。

    • 当你运行 docker pull 时，Docker 会首先获取镜像的 manifest，然后根据 manifest 下载对应的镜像层。
    • 如果 manifest 不存在或无法访问，Docker 会返回 manifest unknown 错误。

    2. 错误原因

    • foundationdb/build:latest 不存在：
      • 可能 FoundationDB 官方没有提供 latest 标签的镜像。
      • 或者镜像仓库中没有 foundationdb/build 这个镜像。
    • 镜像仓库问题：
      • 镜像仓库可能无法访问，或者镜像已被删除。

    3. 解决方法

    （1）检查镜像是否存在

    访问 Docker Hub 查看 foundationdb/build 镜像是否存在：
