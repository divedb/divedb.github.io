​ Linux内核在运行时会生成日志。这些日志对于系统管理、故障排除和性能分析至关重要。

2. 内核日志的来源

​ Linux内核使用printk()函数来生成日志信息。这个函数类似于C语言的printf()，但专门用于内核空间。内核中的各种事件，例如硬件驱动程序的初始化、错误、警告、系统调用等，都会通过printk()记录到日志中。

内核日志的级别:

内核日志信息有不同的级别，表示其重要性。级别越低（数值越小），优先级越高。以下是一些常见的日志级别：

  • KERN_EMERG(0)：系统崩溃等紧急情况，需要立即采取行动。
  • KERN_ALERT(1)：需要立即采取行动的严重问题。
  • KERN_CRIT(2)：严重错误。
  • KERN_ERR(3)：错误情况。
  • KERN_WARNING(4)：警告情况。
  • KERN_NOTICE(5)：普通但需要注意的情况。
  • KERN_INFO(6)：信息性消息。
  • KERN_DEBUG(7)：调试信息。

如何查看内核日志:

  1. dmesg命令：dmesg命令用于显示内核环缓冲区的内容，其中包含了内核启动以来的日志信息。这是查看内核日志最常用的方法。
    • dmesg：显示所有内核日志。
    • dmesg -H：以更易读的格式显示日志。
    • dmesg -l <level>：只显示指定级别或更高级别的日志，例如dmesg -l err只显示错误信息。
    • dmesg -w：实时监控内核日志。
  2. /proc/kmsg文件：/proc/kmsg是一个伪文件，提供了对内核环缓冲区的直接访问。可以使用cat /proc/kmsg或tail -f/proc/kmsg等命令来查看日志。
  3. /var/log/messages（或/var/log/syslog）：一些Linux发行版会将内核日志记录到/var/log/messages或/var/log/syslog文件中。这通常由rsyslog或syslog-ng等系统日志守护进程处理。

控制内核日志的输出级别:

/proc/sys/kernel/printk文件控制着内核日志的输出级别。该文件包含四个值：

  • 控制台日志级别：只有级别高于此值的消息才会被打印到控制台。
  • 默认消息日志级别：如果没有指定日志级别，则默认使用此级别。
  • 最低的控制台日志级别：控制台日志级别不会低于此值。
  • 默认控制台日志级别：在引导时设置的控制台日志级别。

可以使用cat /proc/sys/kernel/printk查看当前设置，使用echo "值1值2值3值4" > /proc/sys/kernel/printk修改设置。例如，echo "1417"> /proc/sys/kernel/printk将控制台日志级别设置为1(KERN_ALERT)，默认消息日志级别设置为4(KERN_WARNING)。

input_iterator 和 forward_iterator 是两种迭代器类别，定义于标准库的迭代器概念中。它们的主要区别在于功能和适用的场景：

1. 共性

  • 基本功能：两者都可以顺序访问容器中的元素。
  • 单向移动：两者都只支持单向遍历（不能向后移动）。
  • 操作符支持：都支持 operator* 和 operator++。

2. 区别

特性 Input Iterator Forward Iterator
用途 一次性读取数据（通常用于输入流等） 可多次遍历数据（通常用于容器）
可重用性 不保证可重用，迭代一次后可能失效 可重用，支持多次遍历
复制性 不要求支持复制，但可以被移动 必须支持复制，即多个迭代器可同时使用
多次解引用 不保证支持多次解引用 可多次解引用
递增操作 支持 ++，但不保证效率 支持 ++，效率较高
容器适配性 适用于输入流或单次读取场景 适用于标准容器（如 std::vector, std::list）

5. apport

​ apport（application report）是ubuntu中的一个错误报告工具，它用于自动收集崩溃信息并生成报告。该工具在程序崩溃时会捕捉相关数据，包括崩溃的堆栈跟踪、系统状态和用户信息，并可以将这些信息发送给开发者或存储在本地，以便后续分析。

/var目录通常用于存放可变数据文件，即那些在系统运行时可能会变化的文件。具体来说，常见的内容包括：

  1. 日志文件：例如/var/log目录中存放的系统和应用程序日志。
  2. 邮件：如邮件服务器的邮件文件，通常在/var/mail或/var/spool/mail。
  3. 临时文件：某些应用程序在运行时可能会创建临时文件，存放在/var/tmp或/tmp中。
  4. 数据库文件：某些数据库系统可能会将其数据文件存放在/var/lib下。
  5. 缓存：一些应用程序会在/var/cache中存储缓存数据。
  6. 状态文件：例如，应用程序的运行状态信息可能存储在/var/run中。

Numeric库

1. 简介

​ 标准库中的numeric是一个包含数值运算功能的头文件，提供了多个通用的算法和函数模板，主要用于数组、容器或数值的处理。

Common mathematical functions

​ The header cmath provides standard C library mathematical functions such as std::fabs, std::sqrt, and std::sin.

Mathematical special functions (since C++17)

​ The header cmath also provides several mathematical special functions such as std::beta, std::hermite, and std::cyl_bessel_i.

Mathematical constants (since C++20)

​ The header numbers provides several mathematical constants, such as std::numbers::pi or std::numbers::sqrt2

Basic linear algebra algorithms (since C++26)

​ The header linalg provides basic linear algebra algorithms which are based on BLAS.
