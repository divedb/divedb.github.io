    Linux内核源代码情景分析

    发表于 2025-05-18 更新于 2025-05-25 分类于 linux

    ✅ 文件系统核心信息

    字段 类型 含义
    s_list struct list_head 所有超级块组成的全局链表（super_blocks），便于遍历所有挂载的文件系统。
    s_dev dev_t 文件系统所在的块设备号（主设备号+次设备号）。
    s_blocksize_bits unsigned char 块大小对应的位数（如块大小为 4096，则为 12）。
    s_blocksize unsigned long 该文件系统的块大小（如 4096 字节）。
    s_maxbytes loff_t 文件系统中支持的最大文件大小。
    s_type struct file_system_type * 文件系统类型（如 ext4、xfs）结构体指针。
    s_op const struct super_operations * 文件系统对超级块的操作集，如 write_inode、sync_fs。
    dq_op const struct dquot_operations * 磁盘配额操作相关函数集。
    s_qcop const struct quotactl_ops * 用于 quotactl 系统调用的配额控制操作。
    s_export_op const struct export_operations * 用于 NFS 导出的操作。
    s_flags unsigned long 挂载时的参数标志位，如 MS_RDONLY。
    s_iflags unsigned long 内部使用的标志位，SB_I_* 定义在 fs.h 中。
    s_magic unsigned long 文件系统的魔数，用于识别文件系统类型。
    s_root struct dentry * 该超级块对应的根目录 dentry 指针。

    🔐 并发控制和引用计数

    字段 类型 含义
    s_umount struct rw_semaphore 用于卸载操作的读写信号量。
    s_count int 超级块的引用计数。
    s_active atomic_t 活跃计数；用于挂载/卸载判定。

    🔒 安全与加密

    字段 类型 含义
    s_security void * 安全模块使用（如 SELinux）。
    s_xattr const struct xattr_handler ** 扩展属性操作，如 getxattr/setxattr。
    s_cop const struct fscrypt_operations * 文件加密操作指针（fscrypt）。
    s_master_keys struct fscrypt_keyring * 当前挂载使用的主密钥列表。
    s_vop const struct fsverity_operations * fs-verity 相关操作（文件完整性校验）。

    🌐 字符集支持和导出支持

    字段 类型 含义
    s_encoding struct unicode_map * 文件名的编码（如 UTF-8）。
    s_encoding_flags __u16 编码标志，如大小写敏感等。
    s_roots struct hlist_bl_head NFS 导出的备用根目录列表。
    s_mounts struct list_head 当前超级块的所有挂载点链表。

    💽 底层设备信息

    字段 类型 含义
    s_bdev struct block_device * 设备对象指针。
    s_bdev_file struct file * 对应的块设备文件。
    s_bdi struct backing_dev_info * 与页缓存写回等相关的信息结构体。
    s_mtd struct mtd_info * 若为 MTD 设备（闪存），此字段有效。

    💾 磁盘配额支持

    字段 类型 含义
    s_quota_types unsigned int 支持的配额类型位掩码（如用户、组）。
    s_dquot struct quota_info 配额相关配置（如配额启用标志等）。

    📥 写入控制与信息缓存

    字段 类型 含义
    s_writers struct sb_writers 并发写者计数器与同步控制。
    s_fs_info void * 文件系统私有信息（例如 ext4 的 struct ext4_sb_info）。
    s_time_gran u32 时间戳粒度（纳秒）。
    s_time_min/max time64_t 支持的时间戳范围（秒）。

    🔔 事件通知

    字段 类型 含义
    s_fsnotify_mask u32 监听事件掩码。
    s_fsnotify_info struct fsnotify_sb_info * 通知系统相关结构体。

    🆔 标识信息

    字段 类型 含义
    s_id char[32] 用于日志等的可变信息字符串。
    s_uuid uuid_t 文件系统的唯一标识 UUID。
    s_uuid_len u8 UUID 的长度，通常为 16 字节。
    s_sysfs_name char[] 出现在 /sys/fs 下的文件系统标识，不能变。

    🧱 文件系统实现细节

    字段 类型 含义
    s_max_links unsigned int 一个目录中允许的最大硬链接数。
    s_vfs_rename_mutex struct mutex 内核用来保护跨目录重命名。
    s_subtype const char * 文件系统子类型名。
    s_d_op const struct dentry_operations * 默认 dentry 操作。
    s_shrink struct shrinker * 内存回收（shrink）策略。
    s_remove_count atomic_long_t 被删除但仍被引用的 inode 数量。
    s_readonly_remount int 表示正在切换为只读。
    s_wb_err errseq_t 用于 syncfs() 错误报告。
    s_dio_done_wq struct workqueue_struct * 用于异步 I/O 完成后的延迟处理。
    s_pins struct hlist_head 用于锁定不被卸载的数据结构（如 pinning）。

    🧑‍⚖️ 用户命名空间与隔离

    字段 类型 含义
    s_user_ns struct user_namespace * 所属用户命名空间（用于权限隔离）。

    ♻️ LRU 与销毁

    字段 类型 含义
    s_dentry_lru struct list_lru 缓存的 dentry LRU 链表。
    s_inode_lru struct list_lru 缓存的 inode LRU 链表。
    rcu struct rcu_head RCU 回收机制使用的钩子。
    destroy_work struct work_struct 用于延迟销毁超级块。

    🔄 同步与写回

    字段 类型 含义
    s_sync_lock struct mutex 用于 sync_fs() 的序列化锁。
    s_stack_depth int 文件系统堆叠深度（如 overlayfs）。

    📚 inode 管理

    字段 类型 含义
    s_inode_list_lock spinlock_t 保护下方的 s_inodes 链表。
    s_inodes struct list_head 所有 inode 的链表（用于扫描）。
    s_inode_wblist_lock spinlock_t 保护下方 s_inodes_wb 写回链表。
    s_inodes_wb struct list_head 需要写回的 inode 链表。

    inode（index node）是指“类Unix文件系统”中的一种数据结构，用于描述文件系统对象（包括文件、目录、设备文件、socket、管道等）。每个inode保存了文件系统对象数据的属性和磁盘块位置。文件系统对象属性包含了各种元数据（如：最后修改时间） ，也包含用户组（owner）和权限数据

    🌟 基本字段

    struct module *owner;

    • 所属模块，用于内核模块的引用计数，防止在文件使用过程中卸载模块。
    • 通常设为 THIS_MODULE。

    fop_flags_t fop_flags;

    • 标志位，用于标识这个 file_operations 实例的属性，如是否是非阻塞、安全、支持异步等。
    • 较新版本内核中引入，可忽略旧版本。

    📚 基本文件操作

    loff_t (*llseek)(struct file *, loff_t, int);

    • 文件偏移操作，处理 lseek() 系统调用。

    ssize_t (*read)(struct file *, char __user *, size_t, loff_t *);

    • 用于 read() 系统调用。

    ssize_t (*write)(struct file *, const char __user *, size_t, loff_t *);

    • 用于 write() 系统调用。

    ssize_t (*read_iter)(struct kiocb *, struct iov_iter *);

    • 更现代、支持异步 IO 的读接口（基于 iov_iter 的 vectored IO）。

    ssize_t (*write_iter)(struct kiocb *, struct iov_iter *);

    • 同上，用于 write() 的异步版本。

    🔄 Poll & IO 多路复用

    int (*iopoll)(struct kiocb *, struct io_comp_batch *, unsigned int);

    • 用于 io_uring 的 I/O polling。

    __poll_t (*poll)(struct file *, struct poll_table_struct *);

    • 用于 poll()/select()/epoll() 实现，报告事件状态（如是否可读、可写）。

    📁 目录相关

    int (*iterate_shared)(struct file *, struct dir_context *);

    • 用于列目录（readdir()），多线程安全。

    ⚙️ 控制类接口

    long (*unlocked_ioctl)(struct file *, unsigned int, unsigned long);

    • 实现 ioctl()，用户态控制指令。

    long (*compat_ioctl)(struct file *, unsigned int, unsigned long);

    • 兼容 32 位用户程序的 ioctl 调用（在 64 位内核上）。

    📌 映射与打开关闭

    int (*mmap)(struct file *, struct vm_area_struct *);

    • 实现 mmap() 内存映射。

    int (*open)(struct inode *, struct file *);

    • 打开文件时调用（例如创建 struct file * 时）。

    int (*flush)(struct file *, fl_owner_t id);

    • 在 close() 前刷新数据；注意与 fsync() 区别。

    int (*release)(struct inode *, struct file *);

    • 文件关闭时调用，对应用户态的 close()。

    int (*fsync)(struct file *, loff_t, loff_t, int datasync);

    • 持久化缓存数据到磁盘。

    🔐 文件锁与异步操作

    int (*fasync)(int, struct file *, int);

    • 用于注册异步通知（如 SIGIO）。

    int (*lock)(struct file *, int, struct file_lock *);

    • fcntl() 文件锁接口。

    ⛏️ 其他文件特性支持

    unsigned long (*get_unmapped_area)(...)

    • 为 mmap 分配未映射的地址空间。

    int (*check_flags)(int);

    • 检查 fcntl() 设置的标志是否被支持。

    int (*flock)(struct file *, int, struct file_lock *);

    • 支持 flock 风格的文件锁。

    🧩 splice、copy、remap 等高级操作

    ssize_t (*splice_write)(...)

    • 将 pipe 内容 splice 到文件中。

    ssize_t (*splice_read)(...)

    • 从文件 splice 到 pipe。

    void (*splice_eof)(struct file *file);

    • 当文件达到 EOF 时触发，用于 splice。

    📄 文件空间控制

    int (*setlease)(...)

    • 支持 fcntl() 文件租约机制。

    long (*fallocate)(...)

    • 支持 fallocate() 预分配磁盘空间。

    void (*show_fdinfo)(...)

    • 用于 /proc/<pid>/fdinfo/ 显示调试信息。

    🧠 与内存有关

    #ifndef CONFIG_MMU 下的 unsigned (*mmap_capabilities)(...)

    • 非 MMU 平台下的 mmap 能力描述函数。

    📤 文件复制和重映射

    ssize_t (*copy_file_range)(...)

    • 支持 copy_file_range() 零拷贝文件复制。

    loff_t (*remap_file_range)(...)

    • 将文件区块重映射到另一个文件，类似 reflink。

    💡 文件建议与 IO_uring 扩展

    int (*fadvise)(struct file *, loff_t, loff_t, int);

    • 用于 posix_fadvise()，给内核 IO 策略建议。

    int (*uring_cmd)(...)

    • 用于 io_uring 的自定义命令。

    int (*uring_cmd_iopoll)(...)

    • 上述命令的 polling 版本。

    📝 总结

    类别 成员函数 说明
    基础读写 read / write / llseek 实现基本 IO
    高级 IO read_iter / write_iter / iopoll 异步和 vectored IO
    控制接口 ioctl, mmap, fadvise 管理和控制文件行为
    打开关闭 open, release, flush 生命周期管理
    同步与锁 fsync, lock, flock 数据一致性
    多路复用 poll, fasync 用于 epoll、select
    目录 iterate_shared 目录读取
    splice / copy splice_read, copy_file_range 高效数据移动

    每个进程通过“打开文件”（open()）与具体的文件建立起连接，或者说建立起一个读写的“上下文”。这种连接以一个file数据结构作为代表，结构中有个file_operaitons结构指针f_op。将file结构中的指针f_op设置成指向某个具体的file_operations结构，就指定了这个文件所属的文件系统，并且与具体文件系统所提供的一组函数挂上了钩。

    进程与文件的连接，即“已打开文件”，是进程的一项“财产”，归具体的进程所有。这里有两个指针fs和files，一个指向fs_struct数据结构，是关于文件系统的信息；另一个指向files_struct，是关于已打开文件的信息。

    root所指向的dentry代表着本进程的“根目录”，那就是当用户登陆进入系统时所“看到”的根目录。实际运行时这两个目录不一定都在同一个文件系统中，例如进程的根目录通常是安装于“/”节点上的Ext2文件系统中，而当前工作目录则可能安装于“/dosc”的一个DOS文件系统中。在文件系统的操作中这些安装点起着重要的作用而常常要用到，所以mnt指针就指向代表着这些“安装”的vfsmount数据结构。

    5. 设计优势

    1. 内存效率：
      • 每个文件系统只分配自己需要的空间
      • 避免为最大文件系统（如XFS）预留空间
    2. 模块化：
      • 新文件系统无需修改VFS核心代码
      • 减少内核头文件依赖
    3. 类型安全：
      • 通过 container_of 宏实现安全类型转换
      • 编译时检查类型匹配
    4. 热插拔支持：
      • 动态加载的文件系统模块可以安全管理自己的inode数据
