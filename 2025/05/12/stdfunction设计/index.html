    std::function设计

    发表于 2025-05-12 更新于 2025-05-16 分类于 c++

    std::function是一个通用的函数包装器。它可以存储、复制可拷贝构造和可调用对象（CopyConstructible Callable），包括：

    • 普通函数（通过函数指针）
    • lambda表达式
    • std::bind表达式
    • 成员函数指针
    • 数据成员指针

    存储在std::function中的可调用对象称为目标对象（target）。如果一个std::function没有目标对象，它被称为“空”对象（empty）。 对空的std::function进行调用，将抛出std::bad_function_call异常。

    成员类型

    Type Definition
    result_type R
    argument_type (deprecated in C++17)(removed in C++20) T if sizeof…(Args)==1 and T is the first and only type in Args...
    first_argument_type (deprecated in C++17)(removed in C++20) T1 if sizeof…(Args)==2 and T1 is the first of the two types in Args...
    second_argument_type (deprecated in C++17)(removed in C++20) T2 if sizeof…(Args)==2 and T2 is the second of the two types in Args...

    实现一个类似C++11 std::function的通用函数包装器，主要考虑以下几点：

    1. 类型擦除：能够存储各种可调用对象（函数指针、lambda、成员函数、函数对象等）
    2. 值语义：支持拷贝、移动、赋值等操作
    3. 类型安全：调用时检查参数和返回类型是否匹配
    4. 性能考虑：尽量减少动态内存分配，对小对象进行优化
    5. 异常安全：保证在异常情况下的资源正确释放

    1. c++ named requirements: MoveConstructible

      指明该类型的实例可以从一个右值实参进行构造。

      要求：

      当类型T满足以下条件时，即满足可移动构造（MoveConstructible）要求：

      给定：

      • rv：类型T的右值表达式
      • u：任意标识符

      以下表达式必须有效并产生指定效果：

      表达式 后置条件
      T u = rv; u的值等价于rv初始化前的值。rv的新值是未指定的。
      T(rv) T(rv)的值等价于rv初始化前的值。rv的新值是未指定的。
      注意事项
      1. 类不需要实现移动构造函数即可满足此类型要求：接受const T&参数的拷贝构造函数也可以绑定右值表达式。
      2. 如果可移动构造类实现了移动构造函数，则可以利用构造后rv值未指定的特性来实现移动语义。

      一个可移动构造类意味着满足std::is_move_constructible，但反之则不成立。因为std::is_move_constructible仅检查能否使用正确的参数调用构造函数，而不检查后置条件值。

      is_move_constructible (C++11) 检查类型是否可以从右值引用构造（类模板）

    2. c++ named requirements: CopyConstructible

    指明该类型的实例实例可以通过左值进行拷贝构造。

    要求：

    类型T满足CopyConstructible要求需满足以下条件：

    1. 类型T必须同时满足MoveConstructible（可移动构造）；
    2. 给定以下变量：
      • v：类型为T、const T的左值表达式，或类型为const T的右值表达式；
      • u：任意变量名。

    以下表达式必须合法，且产生指定的语义效果：

    表达式 后置条件（Post-conditions）
    T u = v; u的值与v相等，v的值不变。
    T(v) 构造出的对象的值等于v，v的值不变。

    此外，表达式 v.~T()必须是合法的；对于左值v，表达式&v的类型必须是T*或const T*，且必须计算为v的地址。

    ⛔️（在C++11之前）
    如果一个类重载了operator&，则它将不再满足CopyConstructible，因此不能用于标准库容器。
    这是C++98中的一个设计决策，而不是缺陷（参见LWG issue 390）。

    ✅（从C++11起）
    标准库在需要获取对象地址时，会使用std::addressof，避免因重载operator&导致的问题。

    补充说明：

    如果一个类是CopyConstructible，那么std::is_copy_constructible判断为true，但反过来不一定成立。因为 std::is_copy_constructible只检查语法上能否调用构造函数，并不会检查：

    • 是否满足MoveConstructible；
    • 构造后的对象是否语义等价（后置条件）。
