标准I/O库

发表于 2025-05-13 分类于 文件系统

​ 操作系统提供基本文件访问系统调用（如 open、read 等），但用户在进行文件操作时，往往需要更高级的功能。标准 I/O库（stdio），作为ANSI C标准的一部分，提供额外的功能，避免重复造轮子。

​ 已有大量书籍就少标准I/O库提供的各类调用。而本章则采取不同的视角，通过剖析Linux标准I/O库的实现机制，介绍其核心结构、如何支持各类函数，以及库函数如何映射到底层UNIX系统调用。

应用程序是否使用标准I/O库，而不是直接调用底层系统调用，取决于应用的需求。如果需要更丰富的功能，且对性能要求不高，标准I/O库提供的丰富函数通常能够满足大多数程序员的需要。但如果性能是关键考量，或者需要更精细地控制I/O行为，那么理解文件系统的I/O实现，并绕过标准I/O库直接使用系统调用，会是更优的选择。

本章不会逐一讲解stdio提供的众多函数，而是聚焦于标准I/O库的底层实现机制。关于接口的详细介绍，可参考Richard Stevens所著的《UNIX高级环境编程》一书，或查阅Single UNIX Specification。

FILE结构体

​ 系统调用如open()和dup() 返回一个文件描述符（file descriptor），程序可以通过该描述符进行文件操作。而标准I/O库则基于FILE结构体进行操作，这一结构体也常被称为“文件流”（file stream）。本质上，它是一个字符缓冲区，保存了当前的读写指针及其他辅助信息。

mode参数是一个字符串，指示文件的打开方式，它可以是以下几种模式之一。请注意，这些模式是ANSI C标准的一部分：

  • r, rb：以只读模式打开文件。
  • w, wb：将文件截断为零长度，或者如果文件不存在，则创建一个新文件并以写入模式打开。
  • a, ab：以追加模式打开文件。如果文件不存在，先创建文件。
  • r+, rb+, r+b：以更新模式打开文件（既可读又可写）。
  • w+, wb+, w+b：将文件截断为零长度，或者如果文件不存在，则创建新文件并以更新模式打开。
  • a+, ab+, a+b：以追加模式打开文件。如果文件不存在，创建文件并以更新模式打开。写入将从文件末尾开始。

在内部，标准I/O库将这些模式映射为相应的标志，并将其传递给open()系统调用。例如，r会映射为O_RDONLY，r+会映射为O_RDWR，依此类推。打开文件流的过程如图 4.1 所示。

在Linux上，当程序执行时，动态链接器（ld.so 或 ld-linux.so）会负责加载程序运行所需的动态链接库。这些库包括标准 C 库（libc），它包含了诸如 fopen()、getc() 等函数。在程序执行期间，动态链接器需要读取缓存文件（例如 /etc/ld.so.cache）以快速查找所需的共享库的位置。

即便程序请求每次只读取一个字符，stdio库仍然会尝试从每个文件读取4KB。后续调用getc()不需要再次调用read()，直到缓冲区中的所有字符都被读取。

其他打开文件流的函数

​ 除了fopen()和fclose()，还可以使用以下两个函数来打开文件流：

  • fdopen()：将一个已经存在的文件描述符与文件流关联起来。通常与dup()、pipe()、fcntl()等只返回文件描述符的函数一起使用。
  • freopen()：打开由filename指定的文件，并将stream指向的流与该文件关联。如果原来的流存在，先关闭它。通常用于将文件与标准输入、输出或错误流关联起来。

例如，如果调用者希望使用默认操作在标准输出上执 printf()，但又希望使用另一个文件流作为标准输出，可以通过 freopen()来实现这一目标。

标准I/O库的缓冲

stdio 库使用缓冲机制，旨在最小化对 read() 和 write() 系统调用的调用次数。主要使用以下三种缓冲方式：

  1. 全缓冲：数据将首先写入缓冲区，直到缓冲区满才进行一次大规模的 write() 调用。
  2. 行缓冲：当缓冲区中遇到换行符或缓冲区满时，数据会写入输出设备。
  3. 无缓冲：每次调用都会直接进行 I/O 操作，不使用缓冲区（通常用于错误输出）。

标准 I/O 库的缓冲机制能够有效提高文件 I/O 操作的性能，并减少系统调用的开销。

  • 完全缓冲（块缓冲）

    当字符被写入流时，它们会被缓冲，直到缓冲区满为止。此时，数据会被写入流所引用的文件。类似地，读取操作将尽可能读取整个缓冲区的数据。

  • 行缓冲

    当字符被写入流时，它们会被缓冲，直到遇到换行符为止。此时，包含换行符的整行数据会被写入流所引用的文件。对于读取操作，也是一样，字符会被读取，直到遇到换行符。

  • 无缓冲

    当输出流是无缓冲时，任何写入流的数据都会立即写入与流关联的文件。

ANSI C标准规定，标准输入和输出应该使用完全缓冲，而标准错误则应该是无缓冲的。通常情况下，标准输入和输出在终端设备上是行缓冲的，而在其他设备上则是完全缓冲的。

除了fseek()外，还有以下四个函数可用于处理流中的文件位置：

  • ftell()返回当前文件指针的位置。例如，在调用fseek()之后，ftell()将返回 8192。
  • rewind()相当于fseek(stream, 0, SEEK_SET)，用于将文件指针重置到起始位置。
  • fgetpos()和fsetpos()的功能分别类似于ftell()和fseek()，但它们使用fpos_t类型以支持可移植性和更复杂的文件定位。

小结

​ 标准I/O库提供了大量函数，大大减轻了应用程序开发者的负担。通过尽可能减少系统调用的次数，可以显著提升应用程序的性能。缓冲机制提供了极大的灵活性，使程序员能够更细粒度地控制I/O的行为。

本章介绍了标准 I/O 库的实现方式，但并未涵盖所有相关函数。更多编程视角下的内容可参考Richard Stevens的《Advanced Programming in the UNIX Environment》；而关于ANSI C标准中stdio库的详细描述，则推荐阅读Herbert Schildt的《The Annotated ANSI C Standard》。
