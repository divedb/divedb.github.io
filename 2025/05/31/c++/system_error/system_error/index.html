    system_error

    发表于 2025-05-31 分类于 c++

    该头文件属于C++标准库中的错误处理库（error handling library）的一部分。

    📚 类定义

    • **std::error_category**：错误类别的基类。
    • **std::error_condition**：表示可移植的错误状态码。
    • **std::errc**： 枚举类型，列举了所有标准<cerrno>宏定义对应的错误码。
    • **std::error_code**：表示平台相关的错误码。
    • **std::system_error**：异常类，用于报告包含error_code的错误状态。
    • **std::is_error_code_enum**：类型萃取器，标识某个枚举类型是否为error_code枚举。
    • **std::is_error_condition_enum**：类型萃取器，标识某个枚举类型是否为error_condition枚举。
    • **std::hash<std::error_code>**：为 std::error_code提供哈希支持的模板特化版本。

    🛠️ 概要

    ​ std::error_category是特定错误类别类型（如std::system_category、std::iostream_category等）的基类。每个具体的错误类别类负责定义error_code与error_condition之间的映射关系，并提供相应错误条件的解释。错误类别类的对象通常作为单例使用，并以引用的形式传递。

    🌱 error_condition default_error_condition(int ev)：

    返回给定错误值所对应的错误条件。等价于std::error_condition(val, *this)。

    从error_category派生的类可以重写此函数，用于将特定的错误值映射到通用错误类别。例如，std::system_category重写了该函数，将符合POSIX errno值的错误映射到std::generic_category。

    ​ 通过一个整数值和一个std::error_category唯一标识，但与std::error_code不同，std::error_condition所表示的值不依赖于具体平台。典型的实现方式是：内部保存一个整数值（value）和一个指向std::error_category的指针。

    std::errc是一个作用域枚举，定义了一组与POSIX错误代码对应的可移植错误条件值。

    ​ std::error_code表示一个与平台相关的错误码值。每个std::error_code对象都包含一个来自操作系统或某些底层接口的错误码值，以及指向对应该接口的std::error_category对象的指针。不同错误类别中的错误码值无需保证唯一性。

    🛠️ 函数接口

    • **std::generic_category()**（C++11）
      返回通用错误类别的引用。
    • **std::system_category()**（C++11）
      返回系统级错误类别的引用（如 POSIX errno）。
    • 运算符重载
      • operator==, operator!=, operator<, <=>（C++20 起）
        用于比较 error_code 实例之间的关系。（部分比较运算符在 C++20 中已移除）
    • operator<<
      向输出流中写入错误码的值及其所属类别的名称。
    • **make_error_code(std::errc e)**（C++11）
      将 std::errc 枚举值转换为 error_code 对象。
    • **make_error_condition(std::errc e)**（C++11）
      将 std::errc 枚举值转换为 error_condition 对象。
