    ​ C++标准定义了一组预处理宏，用于标识自C++11起引入的语言和库特性。这些宏旨在提供一种简单且可移植的方式，用于检测特性是否可用。

    属性支持检测

    用于检查当前编译器是否支持名为attribute-token的标准属性（在宏展开之后）。

    对于每个标准属性，__has_cpp_attribute的具体取值由实现自行定义。通常，它会展开为下表中所列的整数值，表示该属性被纳入工作草案的年份和月份（例如 201603 表示 2016 年 3 月）。仅当该属性在当前实现中按照标准推荐行为发挥作用（如触发诊断信息、影响类布局等）时，才会展开为该值；否则，其值为 0。

    对于编译器厂商提供的非标准属性，只要返回非零值即可表示存在支持。

    __has_cpp_attribute 可用于 #if 和 #elif 条件表达式中。在 #ifdef、#ifndef、#elifdef 和 #elifndef（自 C++23 起引入）中也被视为已定义宏。但不能在其他上下文中使用。

    __cpp_aligned_new 是一个 特性宏（feature test macro），用于检测 C++ 编译器是否支持 “对齐的 operator new/delete“ 功能。这个功能是 C++17 引入的。

    🔧 它的作用简述：

    • 宏名：__cpp_aligned_new
    • 类型：特性测试宏
    • 意义：如果定义了，表示你可以使用：
      • operator new(std::size_t size, std::align_val_t align)
      • operator delete(void* ptr, std::align_val_t align)
    • 它允许你显式为某个对象分配 大于默认对齐值（如 16 或 64 字节） 的内存。

    ✅ 具体例子：

    如果你的编译器支持 C++17 并定义了 __cpp_aligned_new，你可以放心使用 new 来分配 alignas(64) 的类型，它会调用对齐版本的 operator new。

    📌 背景与意义

    在 C++11 和 C++14 中：

    • 虽然你可以写 alignas(64) T，但 new T 并不会保证返回的内存是对齐到 64 字节的（最多只保证 std::max_align_t）。
    • 手动使用 _mm_malloc 或 posix_memalign 是唯一选择。
