git checkout

git checkout是一个非常重要的git命令，常用于切换分支和恢复文件。以下是一些常见用法：

1. 切换分支

切换到指定的分支branch-name。

2. 创建新分支并切换

创建一个新的分支new-branch-name并切换到该分支。

3. 恢复文件到某个提交

将指定的文件恢复到某个提交（commit-hash）的状态。

4. 恢复整个工作区

将当前工作区的所有文件恢复到上次提交的状态。

5. 切换到最近的分支

切换回上一个分支。

6. 查看某个提交的内容

查看某个提交的状态，但注意这会让你进入“游离 HEAD”状态。

git checkout到特定的提交而不是分支时，HEAD指针指向某个提交，但不指向任何分支。此时处于一种特殊的状态，称为“游离 HEAD”状态。

特点
  • 没有分支：HEAD 不再指向任何分支，因此在此状态下进行的提交不会被记录到任何分支上。
  • 提交的后果：如果你在这个状态下进行了提交，Git 会创建一个新的提交，但你需要手动将其移动到某个分支上，否则它将孤立。
