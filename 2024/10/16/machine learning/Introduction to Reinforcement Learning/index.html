Introduction to Reinforcement Learning

1. 概念

​ 强化学习（Reinforcement Learning, RL）是一种机器学习的范式，旨在通过与环境的交互来学习如何采取最佳行动，以最大化累积的奖励。其核心思想是通过试错的方式，让智能体在给定的环境中探索和学习。

强化学习的基本要素包括：

  1. 智能体（Agent）: 在环境中做出决策的实体。
  2. 环境（Environment）: 智能体与之交互的外部系统，包括状态、奖励和状态转移。
  3. 状态（State）: 环境在某一时刻的描述，智能体根据状态来做出决策。
  4. 动作（Action）: 智能体在某个状态下可以选择的操作。
  5. 奖励（Reward）: 智能体在执行某个动作后，环境给予的反馈信号，用于评估该动作的好坏。
  6. 策略（Policy）: 智能体选择动作的规则，可以是确定性的（给定状态下选择固定动作）或随机的（根据概率分布选择动作）。
  7. 价值函数（Value Function）: 用于评估在某个状态下的期望收益，帮助智能体判断哪个状态更有利。

2. State

  • Environment state

  • Agent state

  • Markov state

    • 马尔可夫性：当前状态的未来状态只依赖于当前状态，而与过去的状态无关。换句话说，给定当前状态，未来的状态转移不受过去历史的影响。

    • 在强化学习或随机过程的背景下，马尔可夫状态通常指的是系统的状态，该状态能够完整描述系统的未来演变。也就是说，知道当前状态后，未来状态的概率分布只与当前状态相关。

    例子

    • 在棋类游戏中，当前的棋盘布局就是一个马尔可夫状态，因为下一步的可能棋盘布局仅依赖于当前的布局，而与之前的动作和状态无关。
    • 在天气预报中，如果当前状态是“晴天”，那么明天的天气只依赖于“晴天”的状态，而与之前的天气无关。

3. Observability

完全可观测性（Full Observability）
  • 定义：在完全可观测的系统中，所有的状态信息都可以被完全观测到。即，在每个时间点，系统的状态是完全可知的。
  • 特点
    • 代理可以直接获得系统的当前状态信息。
    • 不需要任何额外的推理或估计。
  • 例子：在一个简单的棋类游戏中，玩家可以看到整个棋盘的状态，这就是一个完全可观测的环境。
部分可观测性（Partial Observability）
  • 定义：在部分可观测的系统中，代理只能观察到系统状态的一部分信息，或可能存在噪声，使得状态无法完全确定。
  • 特点
    • 代理必须依赖于观察到的信息来推测未观察到的状态。
    • 需要使用策略来处理不确定性，通常涉及状态估计或历史信息。
  • 例子：在机器人导航中，机器人可能只能感知周围的障碍物，而无法知道环境的完整布局。
应用
  • 在强化学习中，部分可观测环境通常需要使用部分可观测马尔可夫决策过程（POMDP）模型来处理，而完全可观测环境则可以简单地使用马尔可夫决策过程（MDP）模型。

4. Major Components of an RL Agent

  • An RL agent may include one or more of these components:

    • Policy: agent’s behaviour function

    • Value function: how good is each state and/or action

      价值函数（Value Function）是强化学习中的一个核心概念，用于评估某一状态或状态-动作对的“价值”。它帮助代理（agent）决定在不同状态下采取哪些行动，以最大化长期回报。

      主要类型

      1. 状态价值函数（State Value Function, V(s)）：
        • 定义：在给定状态$s$下，代理在该状态下所能获得的预期回报。
        • 表示为：$V(s) = \mathbb{E}[R_t | S_t = s]$，其中$R_t$是从时间$t$开始的累计回报。
      2. 动作价值函数（Action Value Function, Q(s, a)）：
        • 定义：在给定状态$s$和动作$a$下，代理执行该动作后所能获得的预期回报。
        • 表示为：$Q(s, a) = \mathbb{E}[R_t | S_t = s, A_t = a]$。

      在象棋中，假设当前状态是棋盘上某种特定的布局：

      • 状态价值函数（V(s)）：如果在当前布局下，玩家有很高的胜算，比如能确保胜利或导致对手陷入绝境，状态价值函数$V(s)$可以被认为是正值，比如+100，表示这是一个有利的状态。
      • 动作价值函数（Q(s, a)）：假设在该状态下，玩家选择了一步棋（比如移动某个棋子）但这一步棋导致了对手的反击，最终失败。此时，动作价值函数$Q(s, a)$可能是负值，比如-50，表示这一步棋在这个状态下是个不好的选择。

    • Model: agent’s representation of the environment

FAQ

1. state value function $V(s)$ and state-action value function $Q(s, a)$ difference?

​ Bearing that in mind, the definition of these functions should clarify the distinction for you.

$V_{\pi}(s)$ expresses the expected value of following policy $\pi$ forever when the agent starts following it from state $s$.

$Q_{\pi}(s, a)$ expresses the expected value of first taking action $a$ from state $s$ and then following policy $\pi$ forever.

The main difference then, is the Q-value lets you play a hypothetical of potentially taking a different action in the first time step than what the policy might prescribe and then following the policy from the state the agent winds up in.

Example:

That said, lets make the example more concrete. Lets imagine a deterministic grid world. The goal is at (5, 0). As stated in the description, the agent is one step away, so lets say s = (4, 0) for concreteness. For the optimal policy, it will tell us to go right from (4, 0). If we take right, that takes us to our terminating goal state (5, 0), with a single reward of -1 for taking that action after which we stop and receive no more reward. Clearly then, $V^*((4, 0)) = -1$.

As stated in the answer, we want to compare this to the Q-value for an action that takes us one step further away than where we started. Let say $a$ = left.

If we start at (4, 0) and take left, then we’ll end up in (3, 0), with -1 reward.

From here on out we now follow our policy. The agent is now in state (3, 0) and our policy says to take right. That takes us back to where we started: (4, 0) receiving another reward of -1, for a total of -2 reward so far.

But we’re still not done, because we’re only back to where we started! Once again, we follow our policy which from (4, 0) tells us to go right again, now taking us to our terminating goal state (5, 0) with another -1 reward for a total of -3 reward.

So we just accumulated -3 reward; ergo, $Q^*((4, 0), left) = -3$.
