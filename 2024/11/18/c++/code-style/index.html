code style

1. 头文件（Header Files）

  • 每个.cc文件通常应有对应的 .h 文件：

    • 常见规则：每个实现文件对应一个头文件，以便清晰地分离接口和实现。

    • 作用：这种分离便于代码重用、模块化以及独立编译。

    • 例外情况：

      • 单元测试文件：单元测试文件通常用于验证其他模块，不需要单独的头文件。
      • 仅包含 main() 的小型 .cc 文件：这种文件的逻辑是独立的，无需头文件。

  • 正确使用头文件对代码影响显著：

    • 可读性：合理设计头文件能让代码接口更清晰，易于理解。

    • 代码大小：避免不必要的头文件包含，减少编译生成的文件体积。

    • 性能：清理多余的头文件和依赖，可以提升编译效率，降低编译时间。
