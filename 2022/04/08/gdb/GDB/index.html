    4.1 环境变量

    • show paths: 打印搜索路径（就是PATH环境变量值）

    • path directory: 添加搜索路径

    • show environment [varname]: 打印某个环境变量值，environment可以简写为env. 可选参数[varname]不给，输出所有环境变量。varname区分大小写。

    • set environment varname [=value]: 对被调试的程序设置环境变量值。可选参数value不给定，默认为null。

    • unset environment varname: 删除环境变量

    4.2 工作目录

    • pwd: 打印GDB工作目录

    • cd: 改变GDB工作目录

    9. Examine Source Files

    To print lines from a source file, use the list command (abbreviated l). By default, ten lines are printed.

    • list linenum: 展示前后5行

    • list function: 展示函数前后5行

    • list: Print more lines. If the last lines printed were printed with a list command, this prints lines following the last lines printed; however, if the last line printed was a solitary line printed as part of displaying a stack frame, this prints lines centered around that line.

    6. IMUL Instruction

    • The IMUL (signed multiply) instruction performs signed integer multiplication.
    • It has the same syntax and uses the same operands as the MUL instruction.
    • What is different is that it preserves the sign of the product.
    • IMUL sets the Carry and Overflow flags if the high-order product is not just a sign extension of the low-order product.
    • The Overflow flag OF is normally used for interpreting signed arithmetic.

    2. Valgrind检测内存泄漏

    The flags are, in short:

    • --leak-check=full: “each individual leak will be shown in detail”
    • --show-leak-kinds=all: Show all of “definite, indirect, possible, reachable” leak kinds in the “full” report.
    • --track-origins=yes: Favor useful output over speed. This tracks the origins of uninitialized values, which could be very useful for memory errors. Consider turning off if Valgrind is unacceptably slow.
    • --verbose: Can tell you about unusual behavior of your program. Repeat for more verbosity.
    • --log-file: Write to a file. Useful when output exceeds terminal space.
